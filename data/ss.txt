=========================================
=== Справка по ss.
=== инструмент с открытым исходным кодом, который показывает информацию о сетевых сокетах в ОС.
=========================================

Основной синтаксис: ss [опции]. 
Некоторые опции утилиты ss:
-t (--tcp) — показать только TCP-соединения;
-u (--udp) — вывести только UDP-соединения;
-x (--unix) — показать только локальные UNIX-сокеты;
-a (--all) — отобразить все соединения (как активные, так и слушающие порты);
-l (--listening) — показать только службы, которые слушают порты (LISTEN);
-p (--processes) — вывести идентификаторы процессов, использующих порты (требует sudo);
-n (--numeric) — не преобразовывать IP-адреса и порты в имена (ускоряет выполнение);
-s (--summary) — показать статистику по TCP, UDP, ICMP и другим протоколам.

Утилита предоставляет следующую информацию о каждом сокете:
    Netid — тип;
    State — состояние: чаще всего ESTAB (установлен), UNCONN (не подключен), LISTEN (прослушиваемый);
    Recv-Q — количество принятых пакетов в очереди;
    Send-Q — количество отправленных пакетов в очереди;
    Local address:port — адрес локального компьютера и порт;
    Peer address:port — адрес удаленной машины и порт.

1. Показать список всех сокетов:
    $ ss -a

2. Показать список только прослушиваемых сокетов:
    $ ss -l

3. Показать список TCP-соединений:
    $ ss -t

4. Показать список только прослушиваемых TCP соединений:
    $ ss -lt

5. Показать список всех UDP соединений:
    $ ss -ua

6. Показать список только прослушиваемых UDP соединений:
    $ ss -lu

7. Показать информацию о идентификаторах процессов, использующих сокеты:
    $ ss -p

8. Показать информацию о соединениях IPv4 и IPv6:
    $ ss -4
    $ ss -6

9. Показать информацию о соединениях, работающих на определенном порту:
    $ ss -at '( dport = :22 or sport = :22 )'
    $ ss -at '( dport = :ssh or sport = :ssh )'

10. Показать сводную статистику по сокетам:
    $ ss -s

11. Показать сокеты в состоянии TIME-WAIT или CLOSE-WAIT:
    $ ss -tan state time-wait
    $ ss -tan state close-wait

12. Другие полезные комманды:
    $ ss -tulpn
    $ ss -tunap
